- 用户与基础设置模块主要包括用户登录与注册、用户基本信息管理以及系统个性化配置等功能。用户通过登录系统即可使用笔记编辑、双链关联与知识可视化等核心功能，新用户可通过注册建立个人知识空间。登录后的用户能够对个人基本信息进行维护，并根据自身使用习惯对界面显示、编辑器行为及权限扩展等内容进行配置，从而提升系统的个性化使用体验。（未来可能会支持用户自定义CSS样式。） 
- Markdown 编辑与渲染模块是系统的重要基础功能模块，主要用于支持笔记内容的高效编写与实时展示。该模块支持 Markdown 语法的实时解析与渲染，并对数学公式、代码块、图片和视频等内容提供良好支持。同时，系统通过快捷键输入增强编辑效率，并针对双链内容进行渲染，使笔记在编辑状态与阅读状态下均保持良好的一致性与可读性。
-  笔记管理模块主要负责笔记的生命周期管理，包括笔记创建、编辑、删除以及状态维护等功能。用户可以创建新的笔记，对已有笔记进行内容更新或删除操作，同时系统会对笔记的历史版本进行记录，支持用户查看和回溯历史内容。此外，该模块还提供笔记模板与状态管理功能，帮助用户规范笔记结构，提高长期记录与整理效率。 
- 双链与知识关联模块用于实现笔记之间的关联关系构建与维护。用户在编写笔记时，可以通过内部链接方式将不同笔记进行引用，系统会自动生成双向链接关系，并支持反向链接的展示与更新。还具备链接自动补全与提示功能，帮助用户快速建立笔记间的关联结构，从而逐步形成互相连接的知识网络。 
- 知识结构与可视化模块主要用于对笔记及其关联关系进行结构化展示。系统以知识图谱的形式呈现全部或局部笔记节点，直观反映当前笔记之间的关系结构。用户可以查看当前笔记的局部关联图，也可以浏览整体知识网络，并通过图形界面实现节点跳转与知识路径探索，加深对知识体系整体结构的理解。
-  检索与导航模块为用户提供高效的信息定位与内容查找能力。该模块支持全文内容搜索、标题搜索以及标签搜索等多种方式，并可结合筛选条件进行高级检索。通过双链关系的整合，系统还能够辅助用户在不同笔记之间快速导航，提高在大规模笔记库中的查找效率。 
- 导出与同步模块主要负责笔记数据的导出、备份与同步功能。系统支持将笔记内容导出为 PDF 等格式，并通过自动保存机制保障数据安全。实现数据同步，提升系统的可靠性与可持续使用能力

#### 1. 用户注册

##### 接口说明

- 接口路径：POST /register
- 请求方式：POST
- Content-Type: application/json
- 是否需要登录：否

##### 请求参数

| 参数名          | 类型   | 必填 | 说明     | 校验规则       |
| --------------- | ------ | ---- | -------- | -------------- |
| username        | String | 是   | 用户名   | 3-20 个字符    |
| password        | String | 是   | 密码     | 6-20 个字符    |
| confirmPassword | String | 是   | 确认密码 | 必须与密码一致 |
| email           | String | 是   | 邮箱     | 有效邮箱格式   |
| nickname        | String | 是   | 昵称     | 最多 20 个字符 |

##### 请求示例

```json
{
  "username": "testuser",
  "password": "123456",
  "confirmPassword": "123456",
  "email": "test@example.com",
  "nickname": "测试用户"
}
```

##### 响应参数

| 参数名           | 类型    | 说明                |
| ---------------- | ------- | ------------------- |
| code             | Integer | 状态码              |
| message          | String  | 响应消息            |
| data             | Object  | 返回数据            |
| data.id          | Long    | 用户 ID             |
| data.username    | String  | 用户名              |
| data.email       | String  | 邮箱                |
| data.nickname    | String  | 昵称                |
| data.avatarUrl   | String  | 头像 URL            |
| data.status      | Integer | 状态：1 正常 0 禁用 |
| data.createdTime | String  | 创建时间            |
| data.updatedTime | String  | 更新时间            |
| data.token       | String  | JWT 令牌            |

##### 响应示例

```json
成功响应：
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "id": 123456789,
    "username": "testuser",
    "email": "test@example.com",
    "nickname": "测试用户",
    "avatarUrl": null,
    "status": 1,
    "createdTime": "2026-03-20T10:00:00",
    "updatedTime": "2026-03-20T10:00:00",

  }

失败响应 - 用户名已存在：

  "code": 500,
  "message": "用户名已存在",
  "data": null

  "code": 500,
```

- 接口路径：PUT /user
- 请求方式：PUT
- Content-Type: application/json
- 是否需要登录：是
- 认证方式：JWT Token

##### 请求头

| 参数名        | 值             | 必填 | 说明     |
| ------------- | -------------- | ---- | -------- |
| Authorization | Bearer {token} | 是   | JWT 令牌 |

##### 请求参数

| 参数名      | 类型   | 必填 | 说明                     | 校验规则        |
| ----------- | ------ | ---- | ------------------------ | --------------- |
| oldPassword | String | 否   | 旧密码（修改密码时必填） | 6-20 个字符     |
| newPassword | String | 否   | 新密码（修改密码时必填） | 6-20 个字符     |
| nickname    | String | 是   | 昵称                     | 最多 20 个字符  |
| email       | String | 是   | 邮箱                     | 有效邮箱格式    |
| avatarUrl   | String | 否   | 头像 URL                 | 最多 255 个字符 |

##### 请求示例

```json
只修改昵称和邮箱：
{
  "nickname": "新昵称",
  "email": "newemail@example.com",
  "avatarUrl": "https://example.com/new-avatar.jpg"
}

只修改密码：
{
  "oldPassword": "123456",
  "newPassword": "new123456",
  "nickname": "原昵称",
  "email": "原邮箱@example.com",
  "avatarUrl": "原头像 URL"
}

全部修改：
{
  "oldPassword": "123456",
  "newPassword": "new123456",
  "nickname": "新昵称",
  "email": "newemail@example.com",
  "avatarUrl": "https://example.com/new-avatar.jpg"
}
```

##### 响应参数

| 参数名  | 类型    | 说明                        |
| ------- | ------- | --------------------------- |
| code    | Integer | 状态码                      |
| message | String  | 响应消息                    |
| data    | Object  | 返回数据（更新成功为 null） |

##### 响应示例

```json
成功响应：
{
  "code": 200,
  "message": "用户信息更新成功",
  "data": null
}

失败响应 - 旧密码错误：
{
  "code": 500,
  "message": "旧密码错误",
  "data": null
}

失败响应 - 邮箱已被其他账号注册：
{
  "code": 500,
  "message": "邮箱已被其他账号注册",
  "data": null
}

失败响应 - 用户不存在：
{
  "code": 500,
  "message": "用户不存在",
  "data": null
}

失败响应 - 更新失败：
{
  "code": 500,
  "message": "更新失败，请稍后重试",
  "data": null
}
```

---

##### 错误码说明

| 错误码 | 说明                    |
| ------ | ----------------------- |
| 200    | 成功                    |
| 400    | 参数校验失败            |
| 401    | 未授权/Token 无效或过期 |
| 500    | 服务器内部错误/业务异常 |

---

##### 注意事项

1. JWT Token 使用：
   - 登录成功后会返回 token
   - 需要登录的接口必须在请求头中携带 token
   - 格式：Authorization: Bearer {token}
   - token 有效期为 24 小时

2. 密码安全：
   - 密码使用 BCrypt 加密存储
   - 前端传输明文密码即可
   - 修改密码时需要提供旧密码验证

3. 数据校验：
   - 所有字段都有相应的长度和格式校验
   - 校验失败会返回 400 状态码和具体错误信息

4. 邮箱唯一性：
   - 注册时检查邮箱是否被注册
   - 修改时可以修改为自己的邮箱（允许重复提交自己的邮箱）
   - 不能修改为其他用户的邮箱

5. 用户名不可修改：
   - 用户名在注册后不可更改
   - 只能修改昵称、邮箱、头像和密码

2. 知识库与文件夹关系：
   - 一个知识库可以包含多个文件夹和笔记
   - 文件夹支持多级嵌套（父子关系）
   - 笔记可以放在文件夹中或直接放在知识库根目录

3. 删除操作说明：
   - 删除知识库会级联删除其下所有文件夹和笔记
   - 删除文件夹会级联删除其下所有子文件夹和笔记
   - 请谨慎执行删除操作

4. 数据校验：
   - 所有字段都有相应的长度校验
   - 必填字段不能为空
   - 校验失败会返回 400 状态码和具体错误信息

####  分层架构说明

##### Controller 层
- **职责**: 接收 HTTP 请求，参数校验，调用 Service，返回响应
- **规范**: 
  - 使用 `@RestController` 注解
  - 统一返回 `Result<T>` 格式
  - 使用 `@Validated` 进行参数校验
  - 不处理业务逻辑，仅做参数传递和响应封装

##### Service 层
- **职责**: 业务逻辑处理，事务管理，数据校验
- **规范**:
  - 使用 `@Service` 注解
  - 使用 `@Transactional` 管理事务
  - 抛出 `BusinessException` 处理业务异常
  - 复杂业务拆分为多个方法

##### Mapper 层
- **职责**: 数据库操作，SQL 执行
- **规范**:
  - 继承 `BaseMapper<T>`
  - 优先使用 MyBatis-Plus 自带方法
  - 复杂查询自定义 SQL 方法
  - 使用 `@Mapper` 注解

4. 检查邮箱是否重复

1. 接收登录请求（username, password）
2. 查询用户信息

5. 生成 JWT Token
6. 返回用户信息和 Token

1. 解析 JWT Token 获取 userId
2. 验证旧密码（修改密码时）
3. 检查邮箱唯一性
4. 更新用户信息
5. 返回成功

```sql
DROP TABLE IF EXISTS `t_note_link`;

  `anchor_text` VARCHAR(255) DEFAULT NULL COMMENT '链接显示文本',
  `context_snippet` VARCHAR(500) DEFAULT NULL COMMENT '链接上下文摘要（用于展示）',

  `updated_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',

  UNIQUE KEY `uk_t_note_link_source_target_name` (`source_note_id`, `target_note_name`),
  KEY `idx_t_note_link_kb_id` (`knowledge_base_id`),
  KEY `idx_t_note_link_target_note_id` (`target_note_id`),

    FOREIGN KEY (`source_note_id`) REFERENCES `t_note` (`id`)

    ON UPDATE RESTRICT,
  CONSTRAINT `fk_t_note_link_target_note_id`
```
