# 双链答辩演示实例

## 导入方式

1. 先执行 `sql/double_link_notes.sql` 建表。
2. 再执行 `sql/demo_double_link_case.sql` 导入演示数据。
3. 登录系统，账号为 `demo_link`，密码为 `123456`。
4. 进入知识库 `答辩演示知识库`，打开文件夹 `登录认证双链案例`。

## 演示笔记

演示数据包含 13 篇笔记，其中一组围绕 `登录认证模块` 展开，另一组围绕 `个人双链笔记系统总览` 展开。这样答辩时既能说明业务模块和技术点之间的联系，也能说明本系统内部“编辑、解析、存储、图谱展示”之间的关系。

第一组笔记包括：`登录认证模块`、`Spring Security`、`JWT身份认证`、`MyBatis-Plus`、`用户表t_user`、`BCrypt密码加密`、`JwtInterceptor拦截器`、`知识图谱展示`。

第二组笔记包括：`个人双链笔记系统总览`、`笔记编辑模块`、`双链解析机制`、`t_note_link双链关系表`、`Vditor编辑器`。

其中 `登录认证模块` 是答辩时建议重点打开的笔记，它的正文里包含下面几类双链：

```markdown
用户提交账号和密码后，后端基于[[Spring Security]]做安全处理，
使用[[MyBatis-Plus]]查询[[用户表t_user]]中的账号数据，
再通过[[BCrypt密码加密]]校验密码是否正确。

登录成功后，系统生成[[JWT身份认证]]所需的Token，
后续请求会经过[[JwtInterceptor拦截器]]完成身份校验。

如果后续要提高登录速度，还可以考虑加入[[Redis缓存]]。
```

`个人双链笔记系统总览` 这篇笔记可以作为第二个演示入口，它的正文里写入了下面这些双链：

```markdown
系统主要服务于个人知识管理，核心流程包括[[笔记编辑模块]]、
[[双链解析机制]]、[[知识图谱展示]]和[[登录认证模块]]。

用户写笔记时，可以通过[[Vditor编辑器]]输入Markdown内容，
系统会把正文中解析出来的链接保存到[[t_note_link双链关系表]]。
```

## 答辩说明

这个例子可以用来说明双链的作用。传统目录只能说明这些笔记被放在同一个文件夹里，但不能直接说明“登录认证模块”和“Spring Security、JWT身份认证、MyBatis-Plus、用户表t_user、BCrypt密码加密、JwtInterceptor拦截器”之间有什么关系。双链建立后，系统可以从 `登录认证模块` 跳到相关技术笔记，也可以在 `JWT身份认证` 或 `Spring Security` 中看到哪些项目功能引用了它们。

答辩时可以按下面顺序演示：

1. 打开 `登录认证模块`，说明正文中的 `[[Spring Security]]`、`[[JWT身份认证]]`、`[[MyBatis-Plus]]` 等内容会被系统解析成出链；
2. 再打开 `JWT身份认证` 或 `Spring Security`，展示右侧入链中能看到 `登录认证模块`，说明双链可以反向找到应用场景；
3. 打开 `个人双链笔记系统总览`，说明系统总览可以跳转到 `笔记编辑模块`、`双链解析机制`、`t_note_link双链关系表` 和 `Vditor编辑器`；
4. 打开 `知识图谱展示` 页面，说明图谱节点来自笔记，图谱边来自双链关系表；
5. 回到 `登录认证模块`，指出 `[[Redis缓存]]` 是失效双链，说明系统还能记录“已经想到但还没补充”的知识点。

可以这样向老师说明：

“这个例子中，登录认证模块不是孤立的一篇笔记，它和 Spring Security、JWT身份认证、MyBatis-Plus、用户表t_user、BCrypt密码加密都有关系。普通目录只能把这些内容放到同一个文件夹里，双链可以把这种关系记录下来。后期复习 JWT 时，可以看到它被登录认证模块用到了；查看登录认证模块时，也能跳转到相关技术说明。知识图谱则把这些连接变成节点和边，这样能更直观看出知识之间的关系。”
