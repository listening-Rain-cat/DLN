# 第5章 系统实现

> 说明：本章内容基于当前仓库实际实现整理而成，项目真实定位为“基于 Spring Boot + Vue 的个人双链笔记系统”。你前面提供的“社区学习平台”文字主要作为写作形式参考，因此本章在章节组织、图文描述和论文语气上参考毕业论文写法，但功能内容严格以本项目当前代码、数据库和运行结果为准。

## 5.1 系统实现概述

在完成系统需求分析、总体设计和数据库设计之后，需要进一步对系统各功能模块的具体实现进行说明。双链笔记系统采用前后端分离的实现方式，其中后端基于 Spring Boot 框架构建，负责用户认证、知识库管理、笔记存储、双链关系维护、历史版本管理、模板管理、检索与图谱数据提供等业务逻辑；前端基于 Vue 3、TypeScript 和 Vite 构建，负责用户交互界面展示、Markdown 编辑体验、知识结构可视化以及各类业务操作流程的承载。

从整体实现结果来看，当前系统已经形成较为完整的个人知识管理闭环。用户可以通过注册和登录进入系统，在个人知识空间内建立知识库，按文件夹层级组织笔记内容，使用 Markdown 完成正文编写，通过 `[[笔记标题]]` 方式建立双链引用关系，并进一步借助搜索、标签、模板、历史版本和知识图谱等能力提升知识整理与复用效率。

本章主要结合系统运行结果，对用户认证、知识库管理、笔记编辑、双链关联、版本管理、检索导航、模板管理、知识图谱以及个人设置等模块的实现情况进行说明，并给出对应界面截图，以展示系统当前阶段的实际完成效果。

## 5.2 系统实现环境与技术基础

为了保证系统能够稳定运行并满足双链笔记管理、Markdown 渲染、知识关系维护和图谱展示等需求，系统在实现阶段采用了较为成熟的技术组合。其主要运行环境与技术基础如下：

| 类别 | 具体内容 |
| --- | --- |
| 后端框架 | Spring Boot 3.5.11 |
| 安全认证 | Spring Security、JWT |
| 持久层框架 | MyBatis-Plus 3.5.15 |
| 前端框架 | Vue 3、TypeScript、Vite |
| 编辑器 | Vditor |
| 可视化组件 | D3 |
| 数据库 | MySQL 8.0.35 |
| 运行环境 | Java 17、Maven 3.9.12、Node.js 22.14.0、npm 10.9.2 |

在该技术体系下，后端通过 REST 风格接口向前端提供统一的数据访问能力，前端通过统一请求层与后端交互，并将返回结果转换为知识库列表、目录树、笔记详情、检索结果和图谱节点边数据等界面内容。由于系统需要频繁处理笔记正文、双链关系和用户配置，因此数据库设计中采用了“主表 + 扩展表”的方式，将笔记基础信息、正文内容、历史版本、标签关系和双链关系分别存储，以提高结构清晰度和后续扩展能力。

## 5.3 数据库实现情况

数据库是系统实现的重要基础。结合 `sql/double_link_notes.sql` 与当前运行库结构可以看出，系统围绕“用户 - 知识库 - 文件夹 - 笔记 - 双链关系”的核心业务模型进行设计，主要数据表包括用户表、用户设置表、知识库表、文件夹表、笔记表、笔记正文表、历史版本表、双链关系表、模板表、标签表、标签关联表以及附件表等。

其中，`t_user` 表主要保存用户名、邮箱、密码、昵称、头像和账号状态等用户基础信息；`t_user_settings` 表用于保存用户个性化配置，例如内容主题与代码主题；`t_knowledge_base` 表和 `t_folder` 表共同构成知识库的层级组织结构；`t_note` 表保存笔记标题及所属关系，`t_note_content` 表保存 Markdown 原文内容；`t_note_history` 表用于保存历史版本快照；`t_note_link` 表用于维护笔记之间的双链关系；`t_note_template` 表、`t_tag` 表以及 `t_note_tag` 表分别用于模板管理与标签管理。

从结构设计上看，系统数据库具有以下几个特点：

1. 主要业务表主键均采用 `bigint` 类型，并在实体层通过 MyBatis-Plus 的 `ASSIGN_ID` 策略生成雪花 ID，从而保证主键的唯一性。
2. 知识库、文件夹和笔记均设计了 `status`、`delete_token` 和 `deleted_time` 字段，为软删除和后续回收站机制预留了实现基础。
3. 笔记基础信息与笔记正文分表存储，有利于标题和正文的独立维护，也便于自动保存逻辑的实现。
4. 双链关系独立存储在 `t_note_link` 表中，既可以支持出链、入链和失效链判断，也为知识图谱可视化提供了直接数据来源。
5. 标签采用知识库级复用设计，一方面避免同一知识库中重复创建相同标签，另一方面便于用户在多篇笔记之间快速绑定和筛选。

当前本地数据库在 2026 年 4 月 15 日的运行快照如下：

| `t_note` | 19 |
| `t_note_content` | 19 |
| `t_note_history` | 16 |
| `t_note_link` | 10 |
| `t_note_tag` | 5 |
| `t_note_template` | 3 |
| `t_tag` | 6 |

上述结果表明，数据库已经能够支撑系统当前各项核心功能的正常运行。

## 5.5 知识库与目录管理模块实现

用户登录成功后进入知识库工作台。该模块主要用于展示当前知识库基本信息，并提供知识库内容浏览入口。用户可以查看知识库名称、简介、更新时间以及笔记数、模板数、标签数等统计信息，也可以通过资源目录管理当前知识库下的文件夹与笔记。

在界面实现上，系统将知识库工作台与资源目录区域组合展示。上方用于显示当前知识库概览信息，下方用于展示资源目录树。目录节点既可以表示文件夹，也可以表示笔记；文件夹节点支持展开与收起，笔记节点则可以直接点击进入编辑界面。系统演示知识库工作台界面如图 5-3 所示。

![图 5-3 知识库工作台界面](./screenshots/system-implementation/figure-5-3-knowledge-base-workspace.png)

图 5-3 知识库工作台界面

在实现过程中，知识库管理相关逻辑主要由后端 `KnowledgeBaseController` 与 `KnowledgeBaseService` 提供支持。系统已实现知识库的创建、查询、修改和删除操作，同时也实现了知识库目录树组装接口，能够将文件夹和根目录笔记整合为统一的树形结构返回前端。这样一来，前端即可利用递归组件将目录树完整渲染出来，从而形成较为直观的知识组织界面。

## 5.6 笔记编辑模块实现

笔记编辑模块是系统最核心的功能模块之一。用户在资源目录中选择某篇笔记后，即可进入笔记编辑界面。该界面包括标题输入区、正文编辑区、元数据展示区和右侧目录导航区等部分。正文编辑采用 Vditor 编辑器进行实现，能够较好地支持 Markdown 语法输入与渲染。

在实际使用过程中，用户可以直接修改笔记标题和正文内容，系统会根据标题与正文变化自动触发保存逻辑，从而降低手动保存频率并减少数据丢失风险。界面中还展示了当前笔记所属知识库、出链数量、入链数量等信息，便于用户在编辑时同时关注该笔记在知识网络中的位置。双链笔记系统笔记编辑界面如图 5-4 所示。

![图 5-4 笔记编辑界面](./screenshots/system-implementation/figure-5-4-note-editor-and-links.png)

图 5-4 笔记编辑界面

从代码实现来看，笔记编辑模块对应后端 `NoteController` 和 `NoteService`。其中，系统已实现笔记创建、详情查询、标题自动保存、正文自动保存与删除等能力。正文内容被保存在 `t_note_content` 表中，标题和所属关系则保存在 `t_note` 表中，两者共同支撑前端编辑界面的数据加载与更新。

## 5.7 双链关联与历史版本模块实现

双链关联是本系统区别于普通笔记系统的重要特征。在笔记正文中，用户可以使用 `[[笔记标题]]` 的形式引用其他笔记，系统会在保存正文内容时自动解析这类引用，并在数据库中同步维护双链关系。这样不仅可以在当前笔记中查看已存在的链接页面，还能够统计每篇笔记的入链和出链数量，为后续图谱展示和检索导航提供基础数据。

除双链关系外，系统还实现了历史版本管理功能。用户可以手动点击“创建版本”按钮，对当前笔记内容进行快照保存；也可以在“历史版本”窗口中查看该笔记已有版本，并根据需要执行恢复或删除操作。系统通过 `t_note_history` 表保存历史版本的标题、正文、版本号和创建时间等信息，从而实现版本追踪。笔记历史版本界面如图 5-5 所示。

![图 5-5 笔记历史版本界面](./screenshots/system-implementation/figure-5-5-note-history.png)

图 5-5 笔记历史版本界面

这一模块的实现主要依赖 `NoteHistoryController`、`NoteHistoryService` 以及正文解析逻辑。当前版本已经能够满足个人知识整理过程中的版本管理需求，也为后续进一步实现更细粒度的版本比较和恢复机制奠定了基础。

## 5.8 检索与导航模块实现

随着知识库内容不断增加，用户需要更高效的检索方式来快速定位目标笔记。为此，系统实现了知识库范围内的检索与导航模块。该模块支持按照关键词、目录、标签和检索范围等条件进行组合查询，其中检索范围包括综合检索、标题检索、正文检索和标签检索等多种方式。

在检索结果展示上，系统会显示笔记标题、所属目录、命中摘要、标签信息以及入链、出链、失效链数量等附加数据。这样可以使用户不仅找到目标笔记，还能快速判断该笔记在知识网络中的关联情况。系统检索与导航界面如图 5-6 所示。

![图 5-6 检索与导航界面](./screenshots/system-implementation/figure-5-6-search-panel.png)

图 5-6 检索与导航界面

该功能由后端 `GET /knowledgeBases/{knowledgeBaseId}/search` 接口提供支持。前端将用户输入的关键词、目录筛选和标签筛选条件组合提交至后端，后端返回匹配结果列表，前端再将其渲染为检索结果卡片。实际运行结果表明，该模块已经能够较好地满足当前知识库范围内的查找需求。

## 5.9 模板管理模块实现

为了提高用户日常记录的效率，系统实现了通用模板管理模块。模板以用户为单位进行维护，属于跨知识库复用资源。用户可以根据自己的写作场景建立固定结构的模板，例如周学习记录、会议纪要、阅读笔记或项目复盘等，在新建笔记时直接套用，从而减少重复输入。

在模板中心界面中，系统展示了模板名称、模板说明、模板正文预览以及创建、更新时间等信息，并支持模板的新建、编辑和删除操作。模板管理界面如图 5-7 所示。

![图 5-7 模板管理界面](./screenshots/system-implementation/figure-5-7-template-center.png)

图 5-7 模板管理界面

## 5.10 知识图谱可视化模块实现

为了使用户能够从整体上观察知识点之间的连接关系，系统实现了知识图谱可视化模块。该模块根据双链关系表 `t_note_link` 中的数据，将知识库中的笔记抽象为图谱节点，将笔记之间的引用关系抽象为边，并通过 D3 对节点和边进行可视化布局。

在图谱界面中，用户可以看到当前知识库中的各笔记节点及其连接关系，同时能够直观地识别某篇笔记的入链和出链强度。当前系统已实现知识库级全局知识图展示，并支持从图谱节点返回目标笔记。知识图谱界面如图 5-8 所示。

## 5.11 个人资料与主题设置模块实现
